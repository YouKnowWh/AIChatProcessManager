# 消息过程日志录制流程

本流程用于录制系统的核心功能：一次消息发送如何被拆分为多类数据块并分别写入数据库表。

## 录制目标

建议录制两段素材：

1. **终端实时日志**
   展示后端在 `send_message()` 过程中依次输出的过程级日志。

2. **管理员前端页面**
   展示后台 `消息过程日志` 页面中同一条链路的结构化展示结果。

## 建议录制顺序

### 一、准备环境

1. 启动后端服务
2. 启动前端服务
3. 用管理员账号登录后台
4. 打开后台页面：`/admin/message-flows`
5. 打开一个普通用户聊天页面，准备发送测试消息

### 二、录制终端实时日志

建议单独录制后端终端窗口，重点展示以下内容：

- `context_loaded`
- `write_user_message`
- `write_user_content`
- `write_ai_message`
- `write_ai_content`
- `write_reasoning`
- `write_tool_call`
- `write_tool_result`
- `write_metadata`
- `write_log`

推荐录制动作：

1. 将后端终端窗口置于前景
2. 开始录屏
3. 在前端聊天页发送一条测试消息
4. 保持终端窗口可见，等待日志完整滚动
5. 录制结束

### 三、录制管理员页面

建议录制后台 `消息过程日志` 页面，重点展示：

- 最新 `Trace`
- 每一步对应的 `table`
- `summary`
- `extra` 中的关键字段

推荐录制动作：

1. 将后台 `消息过程日志` 页面置于前景
2. 开始录屏
3. 在用户端发送一条新消息
4. 切回后台页面，等待最新链路自动刷新出现
5. 展开并滚动查看 `steps`
6. 录制结束

## 测试消息建议

为了确保能触发工具调用和元数据写入，建议发送包含检索意图的问题，例如：

`请帮我检索 AI 会话过程管理系统的核心数据块，并总结每一块对应的作用`

## 成片建议

最终可在 PPT 中使用两种方式：

1. **只放终端录屏**
   更强调工程实现与真实过程。

2. **终端录屏 + 后台页面录屏**
   更强调“过程产生”和“治理可视化”两端闭环。
